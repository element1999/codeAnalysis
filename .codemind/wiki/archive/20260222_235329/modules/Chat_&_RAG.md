# Chat & RAG 模块文档


## 1. 模块概述
### 职责
提供**聊天界面**与**检索增强生成（RAG）** 功能，实现用户与知识库的交互式对话体验。通过结合嵌入存储（Embedding Storage）与大型语言模型（LLM）的能力，为用户提供知识增强的回复。

### 定位
作为系统中的**用户交互核心模块**，负责处理用户输入、检索相关知识、生成响应，并管理对话状态。是连接前端界面与后端知识检索/LLM能力的桥梁。

### 设计意图
- 实现知识增强的对话体验，避免LLM“幻觉”问题；
- 封装复杂的RAG流程（检索、排序、融合），简化上层调用；
- 支持会话状态管理，保持对话上下文连贯性；
- 提供可扩展的接口，便于集成不同LLM或嵌入模型。


## 2. 包含文件
| 文件路径                     | 作用描述                                                                 |
|------------------------------|--------------------------------------------------------------------------|
| `codemind/chat/base.py`       | 定义聊天模块的基础接口（如 `ChatBase` 抽象类）和通用工具函数。               |
| `codemind/chat/manager.py`    | 核心管理类（如 `ChatManager`），负责会话生命周期管理、RAG流程协调。          |
| `codemind/chat/rag.py`        | RAG检索增强的具体实现，包括知识检索、排序、融合逻辑。                       |
| `codemind/chat/__init__.py`   | 模块初始化文件，导出核心类/函数，简化上层导入。                             |


## 3. 核心功能
- **聊天对话管理**：支持多轮对话，维护会话上下文（如历史消息、用户状态）；
- **RAG检索增强**：从嵌入存储中检索相关知识，结合LLM生成更准确的回复；
- **嵌入存储交互**：通过嵌入模型将知识库转化为向量，支持高效检索；
- **LLM集成**：调用LLM生成回复，支持自定义模型（如OpenAI、本地LLM）；
- **会话状态管理**：保存/加载会话状态，支持断点续聊。


## 4. 关键组件
### 4.1 核心类
| 类名              | 所属文件       | 作用                                                                 |
|-------------------|---------------|----------------------------------------------------------------------|
| `ChatBase`        | `base.py`     | 聊天模块的抽象基类，定义核心接口（如 `chat`、`reset` 方法）。             |
| `ChatManager`     | `manager.py`  | 聊天管理核心类，协调RAG流程、会话状态管理，是上层调用的主要入口。         |
| `RAGRetriever`    | `rag.py`      | RAG检索器，负责从嵌入存储中检索相关知识，支持多种检索策略（如Top-K）。    |

### 4.2 关键函数
| 函数名               | 所属文件       | 作用                                                                 |
|----------------------|---------------|----------------------------------------------------------------------|
| `init_chat_manager`  | `manager.py`  | 初始化 `ChatManager` 实例，配置RAG参数（如嵌入模型、LLM配置）。           |
| `retrieve_knowledge` | `rag.py`      | 执行知识检索，返回与用户查询相关的知识片段。                             |


## 5. 使用方法
### 5.1 基本流程
1. **初始化聊天管理器**：配置RAG参数（嵌入模型、LLM、知识库路径等）；
2. **启动会话**：调用 `chat` 方法处理用户输入，获取增强后的回复；
3. **管理会话状态**：通过 `save_session`/`load_session` 保存/加载会话状态；
4. **重置会话**：调用 `reset` 方法清空上下文，开始新对话。

### 5.2 示例代码
```python
from codemind.chat.manager import init_chat_manager

# 1. 初始化ChatManager
chat_manager = init_chat_manager(
    embedding_model="text-embedding-ada-002",  # 嵌入模型
    llm_config={"model": "gpt-3.5-turbo"},     # LLM配置
    knowledge_base_path="./data/knowledge.db"  # 知识库路径
)

# 2. 处理用户输入
user_query = "什么是RAG？"
response = chat_manager.chat(user_query)
print(response)  # 输出：知识增强的回复

# 3. 保存会话状态（可选）
chat_manager.save_session("session_1.pkl")

# 4. 重置会话（可选）
chat_manager.reset()
```

## 总结
`Chat & RAG` 模块通过封装RAG流程和会话管理，为上层应用提供了简洁的聊天交互接口。其设计注重可扩展性，支持自定义嵌入模型、LLM和知识库，适用于知识问答、客服系统等场景。
